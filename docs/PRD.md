## 1. 文档说明

### 1.1 文档目标
本 PRD 旨在完整描述 KSGC（智码云）平台的产品需求，使第三方研发团队或 AI 工具能在不依赖原代码的情况下，复刻该平台的所有页面、功能与交互逻辑。

### 1.2 阅读约定
- 所有金额单位为「元（CNY）」
- 所有 Token 数量单位为「个」，超大数值使用 K（千）/ M（百万）/ B（十亿）展示
- 所有时间默认为本地时区（UTC+8）
- 「✅必须」「⚠️可选」「❌禁止」用于标识需求强度
- 所有数据为前端 mock，**无需真实后端**，但接口/数据结构需保留以便后续接入

### 1.3 复刻范围
本系统是**前端单页应用（SPA）**，所有数据使用 mock 数据驱动，不涉及实际的后端服务器、数据库或外部 API 调用。复刻者需要构建：
- 完整的页面 UI
- 模拟的数据交互（增删改查可走内存状态）
- 完整的路由跳转与表单交互

## 2. 产品概述

### 2.1 产品定位
**KSGC（智码云）** 是一款面向企业的 AI Coding 辅助服务云平台。它为企业提供：
- 统一的多模型 AI 编码服务接入
- 精细化的成员/部门/模型权限管理
- 全面的用量与性能监控
- 企业级安全（IP 白名单、SSO、审计日志）

### 2.2 业务模式
- **客户**：付费企业（B2B）
- **运营方**：KSGC 平台运营团队，通过 USS 后台管理多家客户
- **终端用户**：客户企业内的开发者，通过 CLI 客户端使用 AI 编码能力

### 2.3 核心价值
| 角色 | 价值 |
|------|------|
| 企业管理员 | 集中管理 AI 服务、控制成本、保障安全 |
| 平台运营 | 多客户全局观察、配置定价倍率、监控基础设施 |
| 开发者 | 查看个人用量、配置 Skills 和 MCP |

## 3. 用户角色与场景

### 3.1 角色矩阵

| 角色 | 模块入口 | 主要职责 |
|------|----------|----------|
| 平台运营 | USS | 客户管理、模型可见性、积分倍率、全局统计 |

| 潜在客户 | Landing + Onboarding | 浏览产品、试用注册、订单确认 |

### 3.2 典型场景

**场景 A：平台运营查看某客户用量异常**
1. 进入 USS → 客户管理
2. 在客户列表筛选异常客户
3. 进入客户详情 → 使用统计标签
4. 调整时间范围、模型筛选，查看趋势/错误日志

**场景 B：企业管理员开通新模型**
1. 进入 Console → 模型管理
2. 启用模型 → 弹出 API 服务协议 → 勾选同意
3. 模型生效后，分配给特定部门/成员

**场景 C：运营方配置客户可见模型与限速**
1. 进入 USS → 客户详情 → 配置 Tab
2. 打开模型配置弹窗
3. 切换模型可见性
4. 对每个可见模型配置 RPM/TPM（含总量、已分配、本次分配 三段展示）

## 5. 指标与术语定义

> 本节为产品的"业务字典"，所有图表、表格、计算都需遵循。

### 5.1 基础指标

| 缩写 | 全称 | 定义 |
|------|------|------|
| TPM | Tokens Per Minute | 每分钟 Token 数 = 输入 token + 输出 token |
| RPM | Requests Per Minute | 每分钟请求数 |
| TTFT | Time To First Token | 单个请求中，从发送到首个输出 token 返回的时延 |
| P98 TTFT | 98% Percentile TTFT | 按小时分桶，每小时 TTFT 排序后第 98% 位 |
| TPOT | Time Per Output Token | Token 生成速度 = 总生成 token / 总生成耗时 |

### 5.2 表格指标计算规则

| 指标 | 计算公式 |
|------|----------|
| TPM（日均） | 当日 token 总量 / 1440；查询时段 < 1 天则 token 总量 / 查询时段分钟数 |
| TPM（峰值） | 查询时段内最高 1 分钟的 token 量 |
| TPM（工作时段） | 9:30–21:30 区间内 token 总量 / 720（min）；查询时段不重叠则显示 `-` |
| TTFT | 查询时段内 ∑首 token 返回时间 / 总请求数 |
| P98 TTFT | <1h：取该时段 98% 位；1~24h：取最高小时 P98；>24h：每小时 P98 中再取 98% 位（剔除未命中缓存 tokens > 128k 的请求） |
| TPOT | 查询时段内 ∑生成 token / ∑生成耗时 |

### 5.3 趋势图粒度规则

| 查询时段 | 数据粒度 |
|----------|----------|
| < 24 小时 | 按分钟 |
| 1 天 ~ 7 天 | 按小时 |
| > 7 天 | 按天 |

- **TPM 趋势**：分钟粒度直出；小时粒度=∑token/60；日粒度=∑token/1440
- **TTFT 趋势**：分钟粒度直出；小时粒度=该小时 ∑首 token 时间 / 请求数；日粒度同理
- **TPOT 趋势**：粒度内 ∑生成 token / ∑生成耗时

## 7. 模块二：Onboarding 入驻流程

### 7.1 流程步骤
分为 4 步，使用步骤条（Stepper）展示进度。

#### Step 1：企业集成配置
- **身份认证源**（单选）：WPS 协作 / 企业微信 / 飞书 / 钉钉
- **配置字段**：App ID、App Key（带显隐切换）、Redirect URI
- 提供「查看配置说明」链接

#### Step 2：云服务资源开通
- 数据库账号（默认 `admin`，可修改）
- 数据库密码（≥8 位，二次确认）

#### Step 3：订阅选择
- **版本**：基础版 / 专业版（卡片式选择）
- **计费方式**：包年包月 / 按量付费 / 试用
- **席位数**：滑块或步进器，范围 20 ~ 2000
- 实时显示价格预估

#### Step 4：订单确认
- 订单概要（版本、计费、席位、总价）
- 支付按钮 → 跳转 `/console`

### 7.2 校验
- 每一步必填项缺失时禁用「下一步」
- 密码两次输入需一致

## 8. 模块三：Console 企业管理控制台

### 8.1 概览（Dashboard）
**信息卡片**：
- 当前订阅计划及到期时间
- 席位使用进度（已用 / 总数）
- 本月 Token 消耗
- 活跃用户数

**快速操作**：
- CLI 安装命令展示（KSGC 官方版 / 自定义终端，带复制按钮）
- 「订阅管理 / 扩容」入口

### 8.2 用量看板

#### 视图切换：全局视图 / 组织视图

**全局视图**：
- 时间范围选择器（今日 / 近 7 天 / 近 30 天 / 自定义）
- 趋势图：活跃用户数、Token 消耗、请求数（复选框可切换）
- 模型消耗排行（横向柱状图）
- 平均耗时统计（按模型）

**组织视图**：
- 三级部门面包屑下钻
- 部门统计表：成员数、活跃数、Token、请求数
- 支持切换到成员维度查看个人详情

### 8.3 调用明细
- **筛选**：时间、模型、调用状态（成功/失败）、成员搜索
- **列表**：时间戳、用户、模型、输入 Token、输出 Token、状态、耗时
- **详情弹窗**：完整输入内容、响应内容、元数据（均支持复制）

### 8.4 模型管理
- 模型列表：模型类型 + 名称、上下文限制、RPM、TPM、详情、启用开关
- 首次启用任意模型 → 弹出《星流平台 API 服务协议》→ 勾选同意才能启用
- 「全部开启」按钮
- 「自动开启新模型」开关
- ✅必须：至少保持 1 个文本模型 + 1 个视觉理解模型启用，否则阻止关闭并 toast 提示

### 8.5 配额管理（按金额）
- 三层级：全局 / 部门 / 成员
- 默认开关关闭，开启后必须输入 > 0 的金额
- 严格层级约束：成员配额 ≤ 部门配额 ≤ 全局配额
- 部门支持多级树形结构

### 8.6 组织成员管理

**成员管理**：
- 成员类型：SSO 同步成员（只读）/ 外部成员（可编辑）
- 状态：正常 / 待激活 / 禁用
- 操作：启用、禁用、编辑（仅外部）、重发密钥（仅外部）
- 模型权限：部门继承（锁定，标"部门继承"）+ 个人额外授予；最终 = 并集；支持拖拽排序

**组织管理**：
- 同步：首次需授权，之后每日 0:00 / 12:00 自动同步，支持「立即同步」
- 树操作：搜索、展开折叠、全选、级联选择
- 批量操作：批量开通/关闭服务、批量配置模型

### 8.7 IP 白名单
- 类型：单个 IP / CIDR 网段
- 校验：仅公网 IP（拒绝 10.x、192.168.x、172.16-31.x）
- 冗余检测：被更大网段包含的规则标记为「冗余」
- 删除需二次确认

### 8.8 系统设置
- **身份源配置**：单选（WPS / 企业微信 / 飞书 / 钉钉）+ App ID / App Key（显隐切换） / Redirect URI；保存后提示用户重新下载 CLI
- **客户端安装包**：上传 .tar.gz / .tgz，自动从文件名解析版本号，版本号唯一；展示安装命令

#### 客户详情

  - 「产品原型」功能开关
  - 「模型配置」按钮 → 打开模型配置弹窗

#### 模型配置弹窗（关键交互）
- 弹窗宽度 `max-w-[1280px]`（≈95vw），`max-h-[85vh]`，`flex flex-col`
- **顶部固定**：标题 + 搜索/筛选
- **中部可滚动**：模型表格（`flex-1 overflow-y-auto`）
- **底部固定 Footer**：「取消」「保存」按钮（保存后 toast `保存成功` 并关闭）

模型表格列（按顺序）：
1. 模型类型（文本/视觉）
2. 模型名称
3. **可见**（Switch 开关，控制该模型对客户是否可见）
4. **RPM 配置**（仅可见模型可编辑）
   - 总 RPM（系统总额，只读）
   - 已配置 RPM（已分配给所有客户的总和，只读）
   - 客户配置（输入框，本次分配；超出剩余额度时输入框 `border-destructive` 红框）
   - 进度条可视化已分配占比
5. **TPM 配置**（同 RPM 三段式：总 / 已配置 / 客户配置）
   - 大数值使用 `formatTPM`（如 2,000,000 → "2M"）
   - RPM 步进 100，TPM 步进 100,000

> ❌ 禁止：模型切换功能开关；开通状态列；可见性放在 RPM 之后

### 9.2 全局统计
两个 Tab：**业务概览** / **模型指标**

**业务概览**：
- 客户活跃排行
- Token / 请求 / 活跃客户的趋势

**模型指标**：
- 各模型 TPM、TTFT、P98 TTFT、TPOT、错误率
- 严格遵循 [5.2 表格指标](#52-表格指标计算规则) 与 [5.3 趋势图粒度](#53-趋势图粒度规则)

### 9.3 积分倍率
- 列表：模型名称、输入倍率（元/K tokens）、输出倍率
- 配置直接影响 My CLI 中显示的消耗金额

### 9.4 模型可见性配置（系统设置）
- 配置每个模型对哪些客户可见
- 配置默认对所有客户可见的模型集合
- 「可见」≠「默认开通」

## 12. 设计系统规范

### 12.1 设计哲学
- 风格：克制、专业、企业级；避免花哨
- 信息密度：偏高，强调表格与图表的可读性
- 暗黑/明亮：以明亮为默认；保留 dark 模式可能性

### 12.2 颜色（HSL 语义 token）
所有颜色 ✅必须 通过 `index.css` + `tailwind.config.ts` 定义为 HSL 语义变量：
- `--background` / `--foreground`
- `--primary` / `--primary-foreground`
- `--secondary` / `--muted` / `--accent`
- `--destructive`（红色，用于超额、错误）
- `--border` / `--input` / `--ring`

❌ 禁止在组件中直接写 `text-white`、`bg-black`、十六进制色值。

### 12.3 字体
- 中文：系统默认（思源黑体 / PingFang SC fallback）
- 数字与英文：等宽数字字体增强表格对齐感

### 12.4 间距与圆角
- 卡片圆角：`rounded-lg`
- 页面留白：`p-6`
- 卡片间距：`gap-4` 或 `gap-6`

### 12.5 图标
统一使用 `lucide-react`。

### 12.6 图表
统一使用 `Recharts`，颜色取自 CSS 变量。

### 12.7 通知
统一使用 `sonner` toast。

## 13. 数据模型

### 13.1 Customer（客户）
```ts
interface Customer {
  id: string;
  companyName: string;
  customerCode: string;        // 客户识别码
```

### 13.2 CustomerDetail（客户详情扩展）

- `modelLatencyTrend[]`：分钟级时延趋势
- `dailyUsage[]`：每日趋势
- `callFailures[]`：失败记录（id、时间、模型、errorCode、errorMessage、requestId、userId、userName）
- `errorCodeDistribution[]`：错误码占比
- `topUsers[]`：活跃用户 TOP
- `recentLogs[]`：审计日志
- `cloudServices`：SLB / 服务器 / 数据库 / Elasticsearch 监控数据
